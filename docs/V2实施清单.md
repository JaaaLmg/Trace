## 2. 硬边界

- 不做 V3 远程 Runner、register、heartbeat、long-poll 或分布式调度。
- 不做 MinIO/S3 artifact storage 或完整 OpenTelemetry。
- replay 只执行冻结测试集，不调用 LLM；`llm_calls` 必须为 0。
- `invalid_test_set`、provider failure、artifact hash mismatch、setup/executor/runtime failure、flaky clean replay 都不能解释成 0 捕获。
- false positive 是 clean baseline 失败，必须和 variant capture 分开。
- real LLM 与 MockLLM 结果分开解释。
- Contract Guard 是系统边界，不是 prompt 装饰。
- prompt 只吃 prompt-safe evidence；audit-only evidence 默认只进 report/audit。
- Dataset readiness 是结构检查，不是 replay 结果，也不是捕获率。
- sampled mutation score 分母只统计 confirmed + probe passed 的 auto mutation variant。
- local_subprocess 是本地进程执行，不是安全沙箱。
- `backend/llm.config.json`、`.env.local` 和任何密钥文件不要动、不要入库。

## 5. 最终验收动作

按 `docs/V2.4_演示runbook.md` 完成一次 API 模式演示：

1. 初始化数据库和 demo benchmark。
2. 启动 API、Worker、前端。
3. 进入 Dataset list 和 Dataset detail。
4. 从 Dataset 创建三策略 Experiment。
5. 在 Experiment detail 启动 `draft` experiment。
6. 等待 completed，检查 Metrics / Evidence / Runtime execution。
7. 下载 metrics JSON，打开 run-level report，并回指 Dataset。
8. 做 cleanup dry-run，记录候选。
9. 做 Runtime preflight / Docker smoke，记录 passed 或 unavailable reason。
10. 把真实证据写回 `docs/V2.4_最终验收报告.md`。
