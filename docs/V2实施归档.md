## 3. 交付范围

### 3.1 评测系统

V2.1 把 V1 的脚本评测升级为 DB experiment：

```text
真实 LLM clean run
-> 冻结 prompt/tool/runtime/strategy snapshot
-> 保存 generated test set artifact
-> clean replay 校验测试集有效性
-> variant replay 只执行冻结测试，不再调用 LLM
-> 聚合 metrics / capture matrix / evidence
-> 前端展示和审计
```

已落地：

- clean run、clean replay、variant replay 分层记录。
- artifact hash 校验。
- provider failure / artifact mismatch / context incomplete 不伪装成 0 捕获。
- `metric_status` 区分可评价、0 捕获、无效测试集。
- `capture_matrix_counts` 表达 repeat 层面的捕获稳定性。
- `cost_per_captured_bug = null` 表达无捕获，不用 0 撒谎。

### 3.2 项目理解与生成约束

已落地：

- source context 从固定 AST 摘要升级为多证据 bundle。
- target source 缺失会阻断 clean run。
- Contract Guard 拒绝弱断言、目标漂移、未知 fixture、2xx-only route oracle、明显矛盾 oracle。
- ReAct Reflection 可消费 pytest failure context。
- EvaluationEvent 可作为 prompt-safe failure context 回灌。

### 3.3 EvaluationEvent 与回灌

已落地稳定事件类型：

- `invalid_test_set`
- `context_incomplete_blocking`
- `flaky_clean_replay`
- `probe_check_passed`
- `probe_check_failed`
- `replay_captured`
- `replay_uncaptured`
- `replay_failure`
- `artifact_hash_mismatch`
- `provider_failure`

口径：

- 事件由现有 DB/artifact/metrics 投影，不新增 DB 表。
- prompt-safe 与 audit-only 分开。
- `provider_failure`、`artifact_hash_mismatch` 默认只审计，不进 LLM prompt。

### 3.4 Auto mutation 最小链路

已落地：

- comparison boundary candidate discovery dry-run v0。
- deterministic sampling：`sample_seed + max_selected`。
- dry-run API 与 audit report CLI。
- selected candidate 人工确认入库。
- confirm 时必须通过 probe clean/buggy 行为校验。
- sampled mutation score 只纳入 probe passed 的 confirmed auto mutation variants。

边界：这不是完整 mutation engine，只是可审计、可扩展的最小入口。

### 3.5 前端闭环

已落地：

- Experiment List / Detail。
- metrics table、capture matrix、clean run drilldown、replay evidence。
- Evaluation Events timeline。
- Experiment 创建入口。
- Dataset 基础创建入口。

未落地：完整 Dataset task/bug/variant 编辑器、Prompt/Runtime/Tool schema 管理页。

## 4. 已化解风险

- 真实 provider 失败曾被评测链路误解释为 0 捕获；已修为 experiment failed。
- clean replay collection/import error 曾污染 capture rate；已修为 `invalid_test_set`。
- target scope 符号解析曾缺目标源码；已修并加 context gate。
- route target / class method / parameterized case 名称曾误判；已归一。
- 真实模型长输出曾被默认 token 上限截断；eval CLI 已支持 `--max-output-tokens`。
- stale env key 曾覆盖 config 文件 key；CLI 真实评测已显式加载 `backend/llm.config.json` / `TRACE_LLM_CONFIG_FILE` 并传入 runtime config。
- `%SystemDrive%/` 本地误产物已加入 `.gitignore`。

## 5. 剩余风险

这些风险不阻塞 V2.1 收工，但会影响 V2.2：

1. subprocess executor 不是安全沙箱。
2. `source_context.py` 仍偏大，后续修改要先拆模块。
3. auto mutation operator 太窄，不能宣传为完整 mutation testing。
4. 真实 LLM 证据只有 demo seeded bug 数据集，不覆盖任意项目泛化。
5. 前端 Dataset / Runtime / Prompt 管理仍不完整。
6. 大规模 replay 的并行、增量和缓存策略未做。
