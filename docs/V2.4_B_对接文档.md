# TRACE V2.4 B 线对接文档

## 1. 分工边界

V2.4 B 线继续负责 Dataset、mutation confirmation、Contract Guard、prompt-safe evidence 和前端演示链路中属于数据集/生成约束的部分。本轮第一刀只收束最终实训/展示验收的可见断点，不开启 V3 平台能力。

本轮没有改变以下硬边界：

- 不做远程 Runner、register、heartbeat、long-poll 或分布式调度。
- replay 仍只执行冻结测试集，不调用 LLM。
- invalid_test_set、provider_failure、artifact_hash_mismatch、setup/executor failure、flaky clean replay 不能算 0 捕获。
- Contract Guard 仍是系统边界，不是 prompt 装饰。
- prompt 只吃 prompt-safe evidence；audit-only 默认只进 report/audit。
- 不修改 backend/llm.config.json、.env.local 或任何密钥文件。

## 2. 本轮新增对接点

### 2.1 LLM 选项只读合同

新增后端只读 API：

- `GET /api/v1/llm-options`

返回内容只包含前端创建实验所需的 provider/model 选择合同：

- MockLLM 选项始终可用，用于确定性演示和 harness 验证。
- 后端运行配置存在真实 LLM provider/model 时，返回一个 configured 选项。
- 真实 LLM 缺少 API key 时标记 `selectable=false`，前端展示但不能选择。
- 响应不返回 `api_key`、`base_url` 明文或任何密钥值，只返回 `base_url_configured` 布尔状态。

该 API 的事实源是后端 LLM runtime config。前端不硬编码真实 provider/model，只消费该 API 返回的可选项并提交 `llm_override`。

### 2.2 Experiment 创建页 provider/model 收口

Experiment 创建页移除裸 provider/model 输入框，改为 LLM option 下拉：

- MockLLM / mock-1 明确显示为 Mock/scripted。
- 后端配置的真实 LLM 明确显示为真实 LLM。
- 不可选项显示 unavailable，不能被提交。
- 创建 experiment 时提交所选 option 的 provider/model override；最终 resolved provider/model 仍以 clean run `strategy_snapshot` 为审计事实。

这解决了 V2.4 演示中“前端随便填 provider/model”的断点，同时不把真实配置文件推给前端。

### 2.3 Dataset 可见入口

顶部导航新增 Dataset 入口：

- demo 数据源直达 `dataset-demo-v2`，方便从 Dataset benchmark 开始演示。
- API 数据源进入现有 Dataset authoring 入口。
- Experiment 列表中的 `dataset_id` 改为可点击入口，能直接跳到对应 Dataset detail。

这只补可见路径，不新增 Dataset schema，不改变 readiness 或 mutation confirmation 口径。

### 2.4 Experiment 详情页启动入口

Experiment detail 页面新增 Start 按钮：

- 仅 API 数据源显示。
- 仅 draft / queued / cancelled 状态可启动。
- 调用既有 `POST /api/v1/experiments/{experiment_id}/runs`。
- 启动后刷新 progress，并保留原有 metrics/evidence 展示路径。

该入口属于 A/B 共同演示链路，但本轮只接前端入口，不改变 experiment runner 状态机。

## 3. A/B 接口约定

- B 线前端可以展示 experiment metrics / EvaluationEvent / prompt-safe 边界，但不解释 A 线 executor 内部 payload。
- LLM option API 只解决创建 experiment 的配置选择，不改变 run 级 `strategy_snapshot` 冻结规则。
- Dataset readiness 继续是结构检查，不是 replay 结果，也不是 captured bug 指标。
- sampled mutation score 分母仍只统计 confirmed + probe passed 的 auto mutation variant。
