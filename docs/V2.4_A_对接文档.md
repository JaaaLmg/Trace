# TRACE V2.4 A 线对接文档

更新日期：2026-06-21

## 1. 当前实际进度

仓库当前已经从 V2.3 进入 V2.4 最终实训/展示验收版。V2.3 A/B 的主体能力已经落地：Dataset authoring、mutation confirmation、Contract Guard、runtime profile、replay/cache/cleanup、artifact inventory、metrics evidence 和前端基础展示链路都已经具备最小闭环。

V2.4 B 线本轮主要完成演示链路和生成约束侧的可见断点：

- 新增 `GET /api/v1/llm-options`，前端创建实验时从后端只读合同读取 MockLLM / configured real LLM 选项。
- Experiment 创建页从裸 provider/model 输入改为 LLM option 下拉，并显示缺失 dataset/runtime/strategy/LLM option 的阻塞原因。
- 顶部导航、Experiment 列表和 Dataset detail 增加 Dataset 可见入口。
- Dataset detail 增加从 Dataset 跳转创建 Experiment 的 query 预选链路。
- Dataset authoring 增加 demo preset、snapshot 下拉、ordinary variant preset。
- Mutation confirmation 增加结构化 probe 表单和只读 Probe JSON 预览。
- Experiment detail 增加 Start 入口并接到既有 `POST /api/v1/experiments/{experiment_id}/runs`。
- Comparison 空状态增加从 Dataset/Experiment 继续演示的路径提示。

A 线本轮已经完成 runtime / replay / metrics / report 侧的最终演示收束：Start 状态机、runtime preflight 明细、cleanup 候选与二次确认、report/export 入口、Comparison 验收口径和 LLM option 后端合同加固均已落地。

## 2. B 线实现风险与改进建议

### 2.1 LLM option 当前是前端收口，不是后端强约束

原风险：`GET /api/v1/llm-options` 已正确避免返回 `api_key` 和 `base_url` 明文，前端也不再裸填真实 provider/model；但 `POST /api/v1/experiments` 曾仍可接收任意 `llm_override.provider/model`。

本轮处理：

- 后端新增 `validate_llm_override_against_options()`，Experiment create 路由会校验 override 必须匹配当前 selectable LLM option。
- MockLLM 仍始终可选；configured real LLM 只有 API key 存在时才可选。
- `llm-options` 配置错误响应改为稳定脱敏码，不再回显原始异常文本。
- 新增测试覆盖坏配置不泄密与不可选 LLM override 拒绝。

保留口径：最终 resolved provider/model 仍以 clean run `strategy_snapshot` 为审计事实；`llm-options` 是创建时合同，不 retroactively 改写历史 experiment。

### 2.2 Experiment detail 的 Start 状态与后端状态机不完全一致

原风险：前端详情页把 `draft / queued / cancelled` 都显示为可 Start；后端 `enqueue_experiment()` 对 `cancelled` 直接返回当前 experiment，不会重新排队。

本轮处理：

- Experiment list/detail 的 Start 入口只对 `draft` 显示。
- `queued/running` 只展示当前状态，不重复启动。
- `cancelled` 不再显示原地 Start，后续如需 rerun 应走 clone/new experiment 合同。

保留口径：V2.4 不实现原地 rerun/clone；failed experiment 仍需新建。

### 2.3 API 模式下 Comparison 失败会回退 demo metrics，容易混淆真实验收证据

原风险：Comparison 在 API 调用失败时会设置 `metrics = demoExperimentMetrics`，容易把 mock fallback 混入真实 benchmark 证据。

本轮处理：

- API 模式下 Comparison 失败不再回退 demo metrics，改为明确错误/空状态。
- Demo 模式仍显示静态 fixture，但不用于 API 验收证据。

保留口径：最终验收截图和报告只能使用 API completed experiment metrics。

### 2.4 LLM config error 文本需要进一步脱敏

原风险：正常配置读取测试已经覆盖“不泄露 api_key/base_url”，但 `get_llm_options()` 曾直接把 `str(exc)` 返回为 `config_error`。

本轮处理：

- `config_error` 改为 `config_file_missing`、`config_parse_failed`、`unsupported_provider`、`config_load_failed` 等脱敏码。
- 详细异常不进入 API 响应。
- 测试覆盖 secret-like 坏配置不泄露。

保留口径：前端只能展示脱敏错误码；排查细节应看服务端日志。

### 2.5 Dataset query 预选缺少“目标不存在”的显式反馈

原风险：`#/experiments?dataset=<dataset_id>` 只在 dataset options 中存在目标 ID 时预选；否则创建页会回退到第一个 dataset。

风险：Dataset detail 跳转在数据刷新、权限或数据库不一致时可能静默落到另一个 dataset，演示时容易创建到错误 benchmark 上。

本轮处理：

- 当前 query dataset 不存在时，Experiment 创建页会展示 warning，提醒重新选择 Dataset。
- 创建表单保留 Dataset 下拉和当前 dataset name/version，避免完全静默落到错误 benchmark。

保留口径：后端仍以创建 payload 中的 `dataset_id` 为事实源；前端 warning 只是演示防误选，不替代后端合同。

### 2.6 Cleanup / cache cleanup 仍缺最终演示保护

原风险：后端 cleanup dry-run、workspace cleanup、replay-cache cleanup 已有安全边界和测试；但 Experiment detail 曾只有四个短按钮，未展示候选清单，也没有执行前二次确认。

本轮处理：

- Experiment detail 将按钮改成 preview/delete 两段式。
- workspace cleanup 和 replay cache cleanup 都必须先 dry-run，展示 candidate 表格后才能 delete。
- delete 前有浏览器二次确认。
- 默认仍 `keep_failed=true`；cache cleanup 继续依赖后端跳过 reused source replay evidence。

保留口径：cleanup 只清理 workspace/cache 文件，不重写历史 metrics。

### 2.7 Report/export 尚未形成最终 UI 闭环

原风险：V2.4 B 文档标注 report/export 仍需 A/B 合并完成；Experiment detail 可看到 metrics、events、clean/replay evidence，但最终报告入口和导出路径不够显式。

本轮处理：

- Experiment detail 新增 Report / Export 区块。
- 支持下载当前 experiment metrics JSON。
- 提供 first clean run report 入口，回到 run-level report。
- 展示 Dataset 回指、metric status 汇总和 report-like artifact 状态。

保留口径：当前仍以 experiment metrics JSON + run-level report 作为 V2.4 最小 report/export 闭环；如后续需要 experiment-level Markdown report，应作为增强项单独实现。

## 3. A 线本轮完成项

### 3.1 后端合同

- Experiment create 会拒绝不在 `GET /api/v1/llm-options` selectable 集合中的 `llm_override`。
- LLM config error 响应脱敏，不回显本地路径、secret-like 字符串或原始配置片段。
- 保持历史 run 的 resolved provider/model 由 clean run `strategy_snapshot` 冻结。

### 3.2 前端演示链路

- Experiment list/detail 的 Start 入口与后端状态机对齐：只对 `draft` 原地启动。
- Runtime 页面展示 executor 能力、isolation/network/resource limit 状态，并展开 preflight checks、replay policy 与 audit-only limitation。
- Experiment detail 展示 workspace cleanup / replay cache cleanup dry-run candidates；delete 需要先预览并二次确认。
- Experiment detail 增加 Report / Export 区块，可下载 metrics JSON、跳转 run-level report、回指 Dataset。
- Comparison API 模式不再使用 demo metrics fallback，避免污染最终验收证据。

### 3.3 B 线对接口径

- B 线 Dataset -> Experiment query 仍只做 UI 预选；A 线创建时继续依赖后端 dataset/runtime/strategy/LLM option 合同。
- B 线 mutation confirmation 的结构化 probe 表单只生成既有 Probe JSON；A 线 sampled mutation score 仍只消费 confirmed + probe passed 的 auto mutation variant。
- B 线 Contract Guard / prompt-safe evidence 继续通过 EvaluationEvent、report quality 与 backfeed audit 展示；A 线不解释 prompt 内部文本。
- B 线 Dataset readiness 继续是结构检查，不作为 replay 结果或捕获率。

## 4. A 线接手边界

A 线后续不需要重做 B 线 Dataset authoring 或 mutation confirmation 体验，但需要消费它们形成最终 benchmark：

- 只把 `confirmed + probe passed + ground_truth.source=auto_mutation` 纳入 sampled mutation score 分母。
- 不把 Dataset readiness 解释成 replay 结果或捕获率。
- 不依赖前端 audit candidate；mutation confirmation 的信任根仍是服务端 rediscovery + probe check。
- replay 继续只执行冻结测试集，`llm_calls=0`。
- local_subprocess 继续只作为本地执行方式，不宣传为安全沙箱。
- API error / fallback / mock 数据不得混入最终真实 benchmark 证据。

## 6. B 线补全审计

对照 `docs/V2实施清单.md` 和 `docs/V2.4_B_对接文档.md`，当前没有阻塞 V2.4 A 线验收、必须再交给 B 线补完的点。

已经对齐的 B 线接口：

- Dataset 可见入口、API Dataset 选择和 demo benchmark 入口已闭合。
- Dataset authoring / mutation confirmation 的最小演示能力已可用，A 线只消费其服务端状态和 probe evidence。
- LLM option UI 与后端 selectable 合同已对齐，A 线创建路由会拒绝不可选 override。
- Contract Guard / prompt-safe / audit-only evidence 的解释仍通过 EvaluationEvent 与 report quality 展示，A 线不消费 prompt 内部文本。

仍属后续增强、不是 V2.4 验收阻塞：

- 新建真实 Dataset 的全自动向导。
- 更完整的 mutation testing 平台。
- experiment-level Markdown report 生成服务。
- cancelled/failed experiment 的 clone/rerun 合同。
- 跨 Dataset / 跨 experiment 聚合对比。
