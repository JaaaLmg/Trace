# V2.2 A 线执行平台说明

## 当前口径

TRACE V2.2 A 线把执行平台拆成可审计 executor 合同。当前支持两类 executor：

- `local_subprocess`：默认执行器，使用本机 Python subprocess 跑 pytest。它有路径约束、独立 workdir、超时和结构化 pytest 插件，但不是安全沙箱。
- `docker`：可选执行器，按 runtime profile 的 Docker image 运行 pytest。Docker 不可用或 image 缺失时必须报告 `executor_unavailable`，不能解释成 0 捕获。

## Runtime Snapshot

每个 run 会冻结 runtime profile 到 `runtime_snapshot`，包括：

- executor kind
- Docker image / working dir
- install/test command
- timeout
- network policy
- resource limits
- artifact / cleanup policy
- executor capabilities
- env key names，不包含 secret value

本地 subprocess 的能力描述必须明确：

- `isolation_level=process`
- `network_enforced=false`
- `resource_limits_enforced=false`
- `workspace_strategy=host_workdir`

因此 UI、metrics 和文档不能把本地 subprocess 称为 sandbox。

## Replay Evidence

每条 replay 现在记录：

- `runtime_snapshot`
- `executor_metadata`
- `workspace_manifest`
- `cache_key`
- `cache_status`
- `source_replay_id`

`workspace_manifest` 是 replay 输入/执行证据的轻量索引，包含 source snapshot、generated test set hash、variant、executor、runtime hash 和 workspace root。

## Cleanup

新增 experiment cleanup dry-run API：

```text
POST /api/v1/experiments/{experiment_id}/cleanup
```

默认 `dry_run=true`，只列出候选 workspace。真正删除时只允许删除 `.pytest_tmp_experiments` 下的 replay workspace，且拒绝删除根目录本身。

## Replay Scheduler

新增 `replay_scheduler` 原语，支持配置并发，并保证返回结果按输入 job 顺序排序。当前主 experiment runner 仍保持顺序执行；后续可以把 variant replay 批次迁入 scheduler，而不改变 metrics 排序口径。

## EvaluationEvent

新增 executor/cache/setup/cleanup 相关事件类型：

- `executor_unavailable`
- `executor_timeout`
- `setup_failed`
- `cleanup_failed`
- `resource_limit_exceeded`
- `network_policy_violation`
- `replay_cache_hit`
- `replay_cache_stale`

这些事件默认是审计证据。是否进入 prompt-safe backfeed 需要单独评审。
