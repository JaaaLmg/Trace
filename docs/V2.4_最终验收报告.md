# TRACE V2.4 最终验收报告

更新日期：2026-06-22

## 1. 结论

V2.4 A/B 合并后已完成最终实训/展示验收所需的演示链路收束。当前版本可以从 Dataset 进入创建实验，在 Experiment detail 启动 draft experiment，并在 completed 后查看 Metrics、Evidence、Runtime、Cleanup 和 Report / Export 入口。

2026-06-21 已在 API 模式生成一条完整 MockLLM 三策略验收实验：`v24-final-api-20260621233841`，状态 `completed`。该实验验证了 Dataset -> Experiment -> Start -> Metrics / Evidence -> Report / Export 的最终展示链路。

本轮没有开启 V3 能力，不包含远程 Runner、分布式调度、对象存储 artifact 或完整 OpenTelemetry。

## 2. 本轮完成

- LLM option 从前端收口升级为后端创建合同校验：`llm_override` 必须匹配 selectable option。
- LLM config error 响应脱敏，不回显 secret、base_url、原始坏配置或本地路径。
- Experiment Start 状态机与后端对齐：只有 draft 可原地启动。
- Runtime profile preflight 展示明细 check、replay policy 和 audit-only limitation。
- Experiment detail 支持 workspace cleanup / replay cache cleanup 预览候选、二次确认和执行结果。
- Runtime evidence、Replay evidence、EvaluationEvent 和 metrics 继续围绕“不把系统失败压成 0 捕获”的口径展示。
- Report / Export 最小闭环完成：metrics JSON 下载、run-level report 入口、Dataset 回指。
- Comparison API 模式不再回退 demo metrics，避免污染真实 benchmark 证据。

## 3. B 线对接

- Dataset readiness 继续是结构检查，不是 replay 结果。
- Mutation confirmation 的前端结构化 probe 只生成既有 Probe JSON，信任根仍是服务端 rediscovery + probe check。
- sampled mutation score 分母只统计 confirmed + probe passed 的 auto mutation variant。
- Contract Guard 仍是系统边界；pipeline reject / invalid test set 不能讲成 prompt warning。
- prompt-safe evidence 与 audit-only evidence 继续按 EvaluationEvent / backfeed policy 区分。

## 4. 指标口径

- replay 不调用 LLM，`llm_calls=0`。
- invalid_test_set 不算 0 捕获。
- provider failure 不算 0 捕获。
- setup/executor/runtime failure 不算 0 捕获。
- artifact hash mismatch 不算 0 捕获。
- flaky clean replay 不算 0 捕获。
- false positive 是 clean baseline 失败，必须和 variant capture 分开。
- zero capture 只用于可评价测试集确实没有捕获 bug 的情况。

## 5. Runtime 边界

- local_subprocess 是本地进程执行，不是安全沙箱。
- Docker Compose 是当前可交付的软件运行路径；Docker executor 不属于 V2.4 验收承诺。
- Docker 不可用时应展示为 executor unavailable 或 preflight failed/warning，不进入捕获率分母。
- cleanup 只清理 workspace/cache 文件，不重写历史 metrics。
